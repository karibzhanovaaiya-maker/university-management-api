## 1. Overview

A layered web application:

| Layer | What | Tech |
|------|------|------|
| Presentation | Static UI + REST API | Next.js (export) · Spring MVC |
| Business | Rules, validation, transactions | Spring `@Service` |
| Persistence | Tables ↔ objects | Spring Data JPA / Hibernate |
| Database | Storage | PostgreSQL |
| Cross-cutting | Auth, errors, docs | Spring Security (JWT), `@RestControllerAdvice`, OpenAPI |

## 2. Architecture

```
Browser ──HTTPS──> Cloudflare ──HTTPS──> nginx :443
                                          ├── "/"                    → static Next.js files (out/)
                                          └── /api · /auth · /swagger → Spring Boot :8080 → PostgreSQL
```

- One public origin → **no CORS**, no separate public port.
- The Java app is a single executable JAR run as a background service (auto-start on boot, auto-restart on crash), listening on internal port 8080.
- PostgreSQL listens on localhost only.

See [uml.html](https://karibzhanova.alma-ai.cc/uml.html) for the deployment, class, ER, sequence and activity diagrams.

## 5. Data model

| Entity | Key fields | Relationships |
|--------|-----------|---------------|
| `User` | id, userName (unique), email, password (BCrypt), role, createdAt | 1:1 ← Student; 1:N → UserFile |
| `Student` | id, studentName, age, groupName, email | 1:1 → User (FK `user_id` unique); M:N ↔ Course |
| `Teacher` | id, teacherName | 1:N → Course |
| `Course` | id, courseName, credits, maxStudents | N:1 → Teacher; M:N ↔ Student |
| `UserFile` | id, fileName, fileType, contentType, storagePath, size | N:1 → User |

Tables: `users`, `students`, `teachers`, `courses`, `student_courses` (join), `user_files`.
All associations are `LAZY`. Schema is generated by Hibernate (`ddl-auto=update`).

## 6. API reference

Base URL: `/` (same origin). Auth: send `Authorization: Bearer <token>`.

### Auth (public)
| Method | Path | Body | Returns |
|--------|------|------|---------|
| POST | `/auth/register` | `{userName, email, password}` | `{token, userName, role}` (role USER) |
| POST | `/auth/login` | `{userName, password}` | `{token, userName, role}` |

### Public
| Method | Path | Notes |
|--------|------|-------|
| GET | `/hello/{name}?lang=` | Lab 1 demo |
| GET | `/public` | open endpoint |

### Courses
| Method | Path | Access |
|--------|------|--------|
| GET | `/api/courses` | any authenticated user |
| GET | `/api/courses/search?page=&size=&sortBy=&direction=` | ADMIN |
| GET | `/api/courses/{id}` | ADMIN |
| POST | `/api/courses` | ADMIN |
| PUT | `/api/courses/{id}` | ADMIN |
| DELETE | `/api/courses/{id}` | ADMIN |

### Users / Students / Teachers (ADMIN)
Same CRUD shape under `/api/users`, `/api/students`, `/api/teachers`
(`GET` list, `GET /search`, `GET /{id}`, `POST`, `PUT /{id}`, `DELETE /{id}`).

### Files (Lab 8)
| Method | Path | Access |
|--------|------|--------|
| POST | `/api/files/{userId}/avatar` (multipart `file`) | authenticated |
| GET | `/api/files/{userId}/avatars` | authenticated |

Full, interactive reference: **Swagger UI** (`/swagger-ui/index.html`). Click **Authorize**, paste a token from `/auth/login`, then **Try it out**.

## 7. Authentication & authorization (Lab 7)

1. `POST /auth/login` → `AuthenticationManager` checks the BCrypt password → `JwtUtil` issues a signed JWT (claims: subject, issuedAt, expiration).
2. The browser stores the token and sends it as `Authorization: Bearer …`.
3. `JwtFilter` (runs once per request) validates the token and sets the authenticated principal + role in the `SecurityContext`.
4. `SecurityConfig` rules: `/auth/**`, `/public`, `/hello/**`, `/swagger-ui/**` are public; `GET /api/courses` is any authenticated user; everything else under `/api/**` is `ADMIN`.
5. Unauthenticated → **401**; authenticated but wrong role → **403**.

## 8. Validation & error handling (Labs 4–5)

DTOs use Jakarta Bean Validation (`@NotBlank`, `@Size`, `@Email`, `@Min`, `@Max`).
`UserFormValidator` adds uniqueness checks (username/email) using a `currentId` to skip the record itself on update.

All errors go through `GlobalExceptionHandler` (`@RestControllerAdvice`) and return a uniform body:

```json
{ "timestamp": "...", "status": 409, "error": "Conflict",
  "message": "Cannot delete teacher with assigned courses", "path": "/api/teachers/1" }
```

| Exception | Status |
|-----------|--------|
| `MethodArgumentNotValidException`, `ValidationException` | 400 |
| `AuthenticationException` | 401 |
| `AccessDeniedException` | 403 |
| `EntityNotFoundException` | 404 |
| `BusinessException` | 409 |

## 9. Filtering, sorting, pagination (Lab 6)

`CourseSpecification.withFilters()` builds a dynamic `WHERE` from optional fields (empty filters → all rows). Sorting is constrained to a **whitelist** of columns to prevent injection; results are paged with `Pageable` and returned as a `Page<…Response>`.

## 11. Running locally

```bash
# toolchain (NixOS): nix-shell      # JDK 17 + Maven
createdb university_db
mvn spring-boot:run                 # API on :8080  (Swagger at /swagger-ui.html)

cd frontend
pnpm install
pnpm dev                            # UI on :3001, proxies /api,/auth to :8080
```

## 12. Testing (Lab 9)

```bash
mvn test
```
JUnit 5 + Mockito unit tests for services (incl. delete-teacher-with-courses → BusinessException, sort whitelist), a JwtUtil round-trip test, and a full `@SpringBootTest` context smoke test against in-memory H2. **17 tests, all green.**

## 13. Deployment

Push to GitHub, then on the server pull + build + restart:

```bash
# backend
cd /srv/university-api && git pull && mvn -DskipTests clean package && sudo systemctl restart university-api
# frontend (static)
cd /srv/university-api/frontend && pnpm install && pnpm run build:static   # nginx serves out/
```

nginx serves the static frontend at `/` and proxies `/api`, `/auth`, `/swagger-ui` to the API. TLS is handled by Cloudflare; the origin uses a self-signed certificate.

## 14. Backup & recovery

```bash
pg_dump university_db > backup.sql     # backup
psql university_db < backup.sql        # restore
```
Source code is versioned in Git, so the whole system can be rebuilt from scratch.
